## 黄金見本（構造・レイアウトの一次参照）

**マスター（マークアップの SSoT）**: 本ディレクトリの **[exemplar-layout.html](exemplar-layout.html)** と **同階層の `images/`**（`learner.png` / `navigator.png`）。見本を改訂するときは **必ずここを編集**し、デプロイ用フォルダ（例: `output/…/index.html`）や Surge へ載せるときは **ここからコピー**する（逆に output だけ直してスキル側を置き換えない）。

**守ること（レイアウト）**

- **ヘッダー**: `header-gradient` + `max-w-4xl` 内の `h1` とリード1行（見本と同じ情報階層）。**内容の正**: [SKILL.md](../SKILL.md) の「リサーチの固定」（1テーマ・選定理由1行・リードの書き方）に合わせる。見本 HTML はあくまで**レイアウト用の文面例**。
- **導入**: **1 つの** `section-card`。学習者ブロック → ナビゲーターブロックの順（両方とも画像アバター + 吹き出し）。見本の `id` は `section-dialogue-intro`。
- **全体の地図**: `h2`（Lucide + 見出し）→ リード文 → `grid gap-4 md:grid-cols-2` のカード（番号付きタイトル + 短い説明）。**意味付けの正**は [SKILL.md](../SKILL.md) の「全体の地図」：**1テーマ用**に **3ブロック以内**、流れは **「定義 → 事業での現れ方（具体例）→ 使い方・次の一歩」** が一目でわかること。カードの**ラベル文言はテーマごとに毎回変える**（仕事／社会／講師の**固定3側面**に対応させる必要はない）。見本はレイアウトとカード数の型を示すだけ。
- **用語**: **1 セクション**に `term-explain` を縦に積む（見本は「先に押さえる用語」+ `lightbulb` の `h2`）。中身の語彙ルールは term-dictionary のみが正。
- **今日の3つの要点**: `check-circle` の **`h2` 文言は「今日の3つの要点」に統一**（旧称「まず押さえる3つ」等は使わない）→ リード → `space-y-4` の行（`badge-essential` / `badge-recommended` と、第3行は見本どおり **`badge-essential` + ラベル「一歩」** や `badge-optional` など、テーマに合わせてよい）+ 見出し + 補足。各行に **一言要約** と **自分の事業での使い道・次の一歩のヒント** を含める（見本の `#section-three`）。**3行の意味（①正体 ②自分の事業と世の中 ③使い方）** の定義は [SKILL.md](../SKILL.md) の表のみを正とする（ここでは繰り返さない）。
- **本編各セクション**（**その1テーマについて**）: `section-card` ごとに `h2`（Lucide + タイトル）→ **学習者対話（`mb-6`）→ ナビゲーター対話（`mb-6`）** → 本文（箇条書き・`rounded-lg border` のまとめ枠・2 カラムのミニ図・`<details>` など、見本にあるパターンから選ぶ）→ 必要なら `text-xs` の参考リンク行。**セクション数の正**: **3〜5**（見本は **`section-misread` / `section-window` / `section-advisor` / `section-next` の 4 本編**でレイアウト例を示す。必要なら増減）。`id` の命名は `section-*` でトピックに合わせてよい。
- **まとめ**: 見本どおり `section-summary` で、**[SKILL.md](../SKILL.md) の「まとめ（最終セクションの必須構成）」**（一言／自分の事業と広い文脈／事業構築のポイント）を**本文または箇条書きで明示**したうえで、学習者の**自分の言葉の要約** → ナビゲーターの肯定・締め（台本の型は dialogue-personas）。
- **出典**: `section-sources` で見出し + 箇条書きリンクの一覧。最後に**免責の1段落**（見本文末の注意書きと同趣旨）。
- **目次**: `main` 内の `id` と `<nav class="toc">` の `href` を一対一で対応。命名は見本のように `section-*` で統一してよい（トピックに合わせてラベル文言だけ差し替え）。

**CSS・スクリプトの骨格**（`:root`、`.section-card`、吹き出し、バッジ、`.toc` の位置など）は [html-structure.md](html-structure.md) の基本テンプレートを正とする。見本 HTML に無いユーティリティ（例: `.code-block`）が必要ならテンプレート側の定義をそのまま使ってよい。**見本と矛盾する独自グリッドや別のカード体系は避ける。**

**`<style>` の二重管理**: 基本テンプレ内の `<style>` と [exemplar-layout.html](exemplar-layout.html) の `<style>` は同一の意味を持つ。配色・コンポーネントを変えるときは **両方を同じ内容に更新**（片方だけ直さない）。

**本スキル本节との優先**（[SKILL.md](../SKILL.md)「本スキルと見本のズレ」に同趣旨）: **コンポーネントの見た目・余白・対話の型**は本ファイル・見本 HTML・html-structure に寄せつつ、**見出し文言・地図ラベル・本編の論点・セクション数**はスキル本节を優先する。

## うまくいく構成（黄金見本と対応する並び）

```
1. ヘッダー（グラデーション）
   └ タイトル + 「この1枚でわかること」一行（＋任意で「今回の1テーマ」と選定理由1行。正は SKILL.md）

2. 導入対話（1 枚の section-card）
   └ 学習者 → ナビゲーター（1テーマに絞る本音 → 短い地図の予告）

3. 「全体の地図」（見本: グリッドカード + 締めのたとえ1段落）
   └ **最大3ブロック**・**定義 → 事業での現れ方（具体例）→ 使い方・次の一歩**の流れがわかること（ラベルはテーマ依存。正は SKILL.md）

4. 用語（1 セクションに term-explain を縦積み）
   └ 「この図解では〇〇と呼ぶ」を必ず入れる（詳細は term-dictionary.md）

5. 今日の3つの要点（h2 文言はこの表記に統一）
   └ 必須 / 推奨 / 一歩（または任意）などバッジで優先度（見本のカード行レイアウトに合わせる）。3行の意味は SKILL.md の表のみを正とする

6. 各トピック section-card（**その1テーマ**について **3〜5** セクションまで）
   └ h2（Lucide）→ 学習者対話 → ナビゲーター対話 → 本文・図・折りたたみ → 参考リンク行（必要に応じて）。見本は4本編（古い記事／窓口／講師・顧問／今日からの一歩）のレイアウト例（※本文は持続化補助のスナップショット。新規図解では事業との結びつき・具体例・誤解しやすい点・次の一歩などに合わせて差し替え）

7. まとめ（section-card）
   └ SKILL.md の必須3点を本文で明示 → 対話（台本・HTML 型は dialogue-personas.md の「まとめ」）

8. 出典一覧（section-card）+ 免責1段落

9. 目次（デスクトップのみ `.toc`）— DOM 位置・マークアップは [html-structure.md](html-structure.md) の「目次（`.toc`）」。リンクは main 内 id と対応（黄金見本の並びを踏襲）
```
